1. 简述C/C++区别

  • 思想上：

    • C：面向过程、函数式，将问题分成不同步骤并调用函数依次实现

    • C++：面向对象、类和对象，将数据和数据操作封装成不同的类，通过创建这些类的对象并调用成员函数来实现对数据模型的操作

  • 应用上：

    • C更偏低层，常常用于嵌入式开发、驱动开发或与硬件交互的领域
    • C++由于优秀的面向对象机制，在大型应用程序开发方面表现出色

  • C++是C的扩充。C++添加的机制有：对象、类模版、异常处理、运算符重载、标准模版库、命名空间、流

2. gcc编译源代码到hello.c的过程

  • 过程：hello.c -> hello.i -> hello.s -> hello.o -> hello

  • 步骤：

    1. 预处理：hello.c -> hello.i
      gcc -E hello.c -o hello.i

      a. 宏定义 #define

      b. 条件编译指令 #ifdef

      c. 处理文件包含指令 #include

      d. 删除注释

      e. 添加行号和文件标识（方便报错）

    2. 编译：hello.i -> hello.s （汇编文件）
      gcc -S hello.i -o hello.s

      a. 词法分析（Lexical analysis）：从左到右读入源程序，按词法规则将源代码分割成一个个单词（Token），输出二元组<单词类别，单词属性>。

      b. 语法分析（Syntax analysis）：识别由词法分析器输出的单词符号序列，构造一棵语法树。语法树指出了词法单元流的语法结构，可以判断是否符合语法规范。

      c. 语义分析（Semantic analysis）：使用语法树和单词符号表中的信息，进一步检查源程序是否满足语言定义的语义约束。

      d. 中间代码生成优化：根据语义分析的输出，生成类机器语言的中间代码，如三地址代码。改进中间代码的质量。

      e. 目标代码生成：将中间代码映射成目标语言(汇编语言)。

    3. 汇编：hello.s -> hello.o （二进制文件）
      gcc -c hello.s -o hello.o

      a. 汇编器将汇编代码翻译成可执行的机器码。每一条汇编语句对应一条机器指令，因此汇编器的工作就是根据汇编指令和机器指令的对照表一一翻译过来即可。

    4. 链接：hello.o -> hello （可执行文件）
      gcc hello.c -o hello

      链接器将多个汇编后得到的机器码文件链接，从而生成一个可执行程序。分为静态链接和动态链接两种方式。

      a. 静态链接：在链接可执行文件时，链接器会找到所有被用到的源代码，将它们复制并组合起来形成一个可执行文件。优点是由于可执行文件中已经具备了执行程序需要的全部内容，因此执行时运行速度较快。缺点是如果有多个可执行程序需要引用同一个目标文件，每个可执行程序都会复制一份该文件的副本，造成空间浪费；另外，如果某个目标文件被修改了，则所有引用该文件的可执行程序都需要重新编译。

      c. 动态链接：程序被拆分成独立的部分存储，只有运行时才链接在一起形成完整的程序。优点是即使多个程序依赖同一个动态链接库，也不需要将这个库复制多份，而是所有程序共享这个库；此外在程序更新时只更新被修改的库，不需要重新链接所有程序。缺点是因为将链接过程推迟到程序运行时期，所以会对程序性能产生损失。

3. C/C++内存管理

  • 内存划分：
    • 栈区：运行时函数分配局部变量、函数参数、返回数据、返回地址
    • 堆区：程序员自动分配
    • 全局区（静态区）：全局变量、静态数据，系统释放
    • 常量区（文字）：存放常量值，系统释放
    • 代码区：存放函数体（类成员函数和全局区）的二进制代码
  • 3种管理数据内存方式：自动存储、静态存储、动态存储
    • 自动存储：函数内部定义常规变量使用自动存储空间，被调用自动产生和消亡（栈）
    • 静态存储：整个程序执行期间都存在的存储方式（static）
    • 动态存储：new/delete，不连续

4. 堆和栈的区别

堆 栈
用户自己申请 系统分配
由不连续的内存块组成的链表，大小可以调整 连续空间，大小固定
速度慢，会产生碎片 速度快，不产生碎片
向高地址增长 向低地址增长
由库函数（malloc/realloc/free, new/delete）提供 由系统提供

5. 什么叫内存泄漏

  • 通常指堆泄漏，程序员向系统申请任意大小堆内存块，使用完毕必须显式内存释放，否则该内存不能再次被使用。（malloc/realloc -> free, new -> delete）
  • 用动态存储分配函数动态开辟的空间，在使用完毕后未释放，结果导致一直占据该内存单元即为内存泄露。
  • 措施：
    1. 使用的时候要记得指针的长度
    2. malloc的时候得确定在那里free
    3. 对指针赋值的时候应该注意被赋值指针需要不需要释放
    4. 动态分配内存的指针最好不要再次赋值
    5. 在C++中应该优先考虑使用智能指针

6. 函数调用过程

int main(void)
{
  ...
  d = fun(a, b, c);
  cout<<d<<endl;
  ...
  return 0;
}

1. main()：

   a. 参数拷贝（push）：c->b->a

   b. 保存 d = fun(a, d, c) 下一条指令：cout<<

   c. 跳转到fun()

   2. fun()：

      a. 移动ebp, esp形成新的栈帧结构

      b. 形成临时变量并执行相关操作（push）

      c. return一个值

      d. 出栈（pop）

      e. 恢复main函数栈帧结构

      f. 返回main()
